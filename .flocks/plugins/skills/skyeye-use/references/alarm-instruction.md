# SkyEye 告警查询参考

> 这里保留旧 skill 中最有用的告警字段与筛选知识，供 `skyeye-use` 在 API 不足或需要对照页面字段时使用。

## 适用范围

- 页面告警筛选字段理解
- API `skyeye_alarm_list` 参数含义理解
- 浏览器回退时对照页面中文字段

## 高频筛选字段

| 页面中文字段 | 接口字段 | 说明 |
| --- | --- | --- |
| 威胁名称 | `threat_name` | 精确或关键字筛选 |
| 威胁级别 | `hazard_level` | 0=低危，1=中危，2=高危，3=危急 |
| 处理状态 | `status` | 0=未处置，1=已处置，6=忽略，7=误报 |
| 数据来源 | `data_source` | 0=全部，1=传感器，2=沙箱，3=邮件沙箱，4=云锁，6=smac，7=蜜罐，8=soar |
| 攻击结果 | `host_state` | -1=失败，0=企图，1=成功，2=失陷 |

## 时间参数

- `start_time`
- `end_time`
- 单位为毫秒时间戳

## 高频示例

最近 7 天高危及以上告警：

```json
{
  "hazard_level": "2,3",
  "offset": 1,
  "limit": 20
}
```

攻击成功告警：

```json
{
  "host_state": "1",
  "offset": 1,
  "limit": 20
}
```

查某个威胁名称：

```json
{
  "threat_name": "暴力破解",
  "offset": 1,
  "limit": 20
}
```

## 返回结果重点关注

- 威胁名称
- 危险等级
- 处理状态
- 数据来源
- 攻击结果
- 受害 / 攻击 IP

## 何时回退浏览器

- 需要页面导出
- 需要复杂联动筛选
- 需要对照页面高级字段而 API 未暴露对应参数
